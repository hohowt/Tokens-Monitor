# AI Token 监控 — 客户端使用说明

本文说明 **Windows 客户端**（`ai-monitor.exe`）的作用、安装方式、配置项与常见问题。面向本机使用者与需要分发给同事的运维同事。

---

## 一、程序是做什么的

本程序在本机启动一个 **HTTP/HTTPS 代理（MITM）**，把经过代理的 **AI 相关 HTTPS 流量**解密后：

- 尽量从响应里解析 **token 用量**（各厂商 JSON 里的 `usage` 等）；
- 将统计结果上报到公司配置的 **服务端**（`config.json` 里的 `server_url`），供大屏汇总与分摊。

**前提**：开发工具（VS Code、Cursor、CLI 等）发出的 AI 请求必须先 **经过本代理**，否则无法统计。当前推荐方式是：用 `ai-monitor.exe --launch <你的程序>` 只对子进程注入代理环境变量，而不是改系统代理。

常见入口已经支持预设：`vscode`、`cursor`、`powershell`、`cmd`，可直接用 `--launch-preset` 启动。若这些程序没加进 `PATH`，客户端还会自动尝试常见 Windows 安装目录。

**不能做什么**：无法保证「未走本代理的流量」被统计；无法替代各厂商官方计费账单；部分 **gRPC / 非 JSON** 场景仅能做体积类 **粗算**（若开启 `report_opaque_traffic`）。

---

## 二、工作原理（简版）

1. 本程序监听 `config.json` 中的 `port`（默认 `18090`），提供 **HTTP 代理**（`CONNECT` 隧道 + 对列入名单的域名做 HTTPS 解密）。
2. 内置一批 **AI 相关域名**（OpenAI、GitHub Copilot、Cursor、Anthropic、OpenRouter 等）；命中则 **MITM** 并尝试解析用量。
3. 未列入的域名一般 **只做透传**，不解密。
4. 需要本机安装本工具生成的 **根证书（CA）**，否则浏览器/IDE 会提示证书错误。

**重要**：本程序是 **HTTP 代理**，**不是 SOCKS5 服务**。在 Proxifier 等软件里请选 **HTTP/HTTPS 代理** 指向 `127.0.0.1:端口`，不要选 SOCKS5 指向同一端口。

---

## 三、环境要求

- Windows 10/11（当前客户端针对 Windows 构建）。
- 能访问公司部署的 **上报服务**（默认示例为 `https://otw.tech:59889`，以你方实际为准）。
- 若使用 **MITM**，需能安装证书到 **当前用户** 证书存储（安装向导或 `安装.bat` 会处理）。

---

## 四、第一次使用（推荐）

1. 将整个 **`client` 文件夹** 拷到本机任意目录（保持 `ai-monitor.exe` 与 `config.json` 在同一目录）。
2. 双击 **`开始使用.bat`**，按向导生成/确认 `config.json`（姓名、部门、`server_url`、端口等）。
3. 向导中建议选择 **「仅安装证书，不改系统代理」**。
4. 向导结束后会启动监控窗口 — **不要关窗口**；再用 `ai-monitor.exe --launch <你的程序>` 启动需要统计的应用。
5. 以后每次开机：再双击 **`开始使用.bat`** 或 **`启动.bat`**（若已配置好 `config.json` 可直接用 **`启动.bat`**）。

如果不想记命令，可以直接双击这些启动器：

- **`启动-VSCode监控.bat`**
- **`启动-Cursor监控.bat`**
- **`启动-PowerShell监控.bat`**
- **`启动-CMD监控.bat`**

## 五、配置文件 `config.json`

可参考同目录下的 **`config.example.json`**。常用字段说明：

| 字段 | 含义 |
|------|------|
| `server_url` | 上报服务根地址，例如 `https://otw.tech:59889` |
| `user_name` / `user_id` / `department` | 大屏展示用身份与部门 |
| `port` | 本机 MITM 监听端口（默认 `18090`）；被占用时程序可能自动顺延到下一可用端口 |
| `upstream_proxy` | 本地 MITM 外联时使用的上游代理，可填 `http://...`、`https://...` 或 `socks5://...`，用于串联公司代理 |
| `install_system_proxy` | 为 `true` 时，执行安装会写入 Windows 系统代理例外等；省略时默认 `false`，不改本机网络 |
| `install_ide_proxy` | 默认 `false`：不往 VS Code/Cursor 的 `settings.json` 写 `http.proxy`，避免与系统代理重复；扩展若完全不走系统代理再考虑改为 `true` 并重新安装 |
| `report_opaque_traffic` | 对难以解析正文的流量是否做体积类粗算上报（可按需关闭） |
| `extra_monitor_hosts` | 额外 **精确主机名 → 供应商标签** |
| `extra_monitor_suffixes` | 额外 **域名后缀 → 供应商标签** |

修改配置后：若改了端口、上游代理或安装模式，请重新执行 **`安装.bat`**（或 `ai-monitor.exe --install`）并重启本程序。

## 六、批处理脚本一览

| 文件 | 作用 |
|------|------|
| `开始使用.bat` | 向导 + 安装 + 启动（适合第一次） |
| `启动.bat` | 仅启动（需已有合法 `config.json`） |
| `启动-VSCode监控.bat` | 一键受管启动 VS Code |
| `启动-Cursor监控.bat` | 一键受管启动 Cursor |
| `启动-PowerShell监控.bat` | 一键受管启动 PowerShell |
| `启动-CMD监控.bat` | 一键受管启动 CMD |
| `安装.bat` | 按当前 `config.json` 安装 CA；仅在 `install_system_proxy=true` 或 `--install-full` 时改系统代理 |
| `快速安装-系统代理.bat` | 快捷调用安装（若项目中有） |
| `重新配置.bat` | 重新跑向导/改配置 |
| `卸载.bat` | 卸载证书与代理相关设置（`--uninstall`） |

## 七、命令行参数（进阶）

在 `client` 目录下于 PowerShell/CMD 中执行，例如：

```text
ai-monitor.exe                    # 正常运行 MITM
ai-monitor.exe --setup            # 仅向导配置
ai-monitor.exe --install          # 默认仅安装 CA，不改系统代理
ai-monitor.exe --install-full     # 强制写系统代理（legacy 模式）
ai-monitor.exe --launch code.cmd  # 仅对子进程注入代理环境变量并启动目标应用
ai-monitor.exe --launch-preset vscode
ai-monitor.exe --launch-preset cursor
ai-monitor.exe --launch-preset powershell
ai-monitor.exe --list-launch-presets
ai-monitor.exe --install-cert-only
ai-monitor.exe --uninstall        # 卸载
ai-monitor.exe -config 路径.json  # 指定配置文件
```

具体以程序 `--help` 输出为准。

## 八、与 Proxifier 共用时的要点

1. **`ai-monitor.exe` 必须「直连 / Direct」**，不要让它自己的出站再进本机代理，否则易 **环路**，导致 Copilot 无响应、上报失败。
2. 指向本机 MITM 时，类型选 **HTTP 代理**，地址 `127.0.0.1`，端口与 **`config.json` 的 `port` 及实际监听一致**。
3. 可勾选 **「通过系统代理连接」**，减少与 Windows 系统代理规则冲突。
4. 避免 **同一连接被代理两次**（系统代理 + Proxifier 重复指向）。

---

## 九、VS Code / GitHub Copilot / Cursor

- 安装好 **CA** 后，Copilot、Cursor 等走 MITM 的 HTTPS 才不会因证书失败。
- 默认推荐用 `--launch` 启动目标程序，让 Electron/CLI 只在当前进程内走 MITM；若确需整机导流，再考虑系统代理模式。
- 若你的同事不想记命令，可优先用 `--launch-preset vscode`、`--launch-preset cursor` 或 `--launch-preset powershell`；即使这些程序未加入 `PATH`，也会优先尝试常见安装目录。
- 客户端新版本已对 **HTTP/2**、**Copilot / Claude 与 OpenAI 不同请求体** 做区分（例如不会对 Anthropic 通道注入 OpenAI 专有字段），请尽量使用 **当前仓库编译的最新 `ai-monitor.exe`**。

---

## 十、自检命令

在监控已启动时（端口以你配置为准）：

```powershell
Invoke-RestMethod "http://127.0.0.1:18090/status"
```

应返回 `status: running`，并带有当前 **`version`**、用户信息等。

## 十一、常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 大屏无数据 | 本机未走代理或监控未开 | 确认窗口在跑、系统/IDE/Proxifier 指向正确端口 |
| 证书错误 | 未安装 CA | 运行 `安装.bat` 或按界面提示手动信任根证书 |
| Copilot 不回复 | Proxifier 环路、协议选错、旧版本 exe | `ai-monitor` 直连、用 HTTP 代理非 SOCKS5、换新版客户端 |
| `/status` 版本很旧 | 实际运行的是别的目录下的旧 exe | 结束旧进程后，在 **当前分发目录** 启动新 `ai-monitor.exe` |

---

## 十二、隐私与安全说明

- MITM 会解密 **列入监控名单的 HTTPS 站点**；请仅在 **公司授权的设备与网络策略** 下使用。
- `config.json` 含上报地址与个人信息，**勿提交到公开仓库**；分发时使用示例配置并替换为内网地址。

---

## 十三、与仓库其他部分的关系

- 仓库中另有 **backend / frontend**（大屏与 API），客户端只负责 **采集与上报**。
- 服务端部署、防火墙与网关策略由运维在服务器侧配置，不在本文展开。
