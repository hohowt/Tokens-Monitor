# Change Log

## [0.1.40] - 2026-04-13

### Fixed
- 恢复扩展默认上报地址为 135，并移除把 135 自动改写到 otw.tech 的迁移逻辑，避免 Cursor 与 VS Code 继续把记录上报到错误服务端

## [0.1.39] - 2026-04-13

### Fixed
- 修复扩展打包时可能捆入未带版本 ldflags 的 ai-monitor.exe，避免安装后的内置客户端显示为 dev 而不是正式版本号

## [0.1.38] - 2026-04-13

### Fixed
- 更新扩展内置的 Windows ai-monitor 代理程序，后续 GitHub Copilot 上报会携带更准确的成本倍率，减少服务端只能按默认折扣回退的情况

## [0.1.34] - 2026-04-12

### Fixed
- 将 Windows 证书自检抽成独立可测试模块，补齐 PowerShell 指纹命中、certutil 兜底、异常分支和证书存储路径字符串回归测试，避免同类误报再次进入发布版本
- 证书自检增加更明确的运行时诊断日志，后续定位宿主环境差异时不再只能依赖面板截图

## [0.1.33] - 2026-04-12

### Fixed
- 修复 Windows 证书自检里 PowerShell 证书存储路径转义错误，避免命令被拼成 Cert:CurrentUserRoot 导致始终误报未受信任

## [0.1.32] - 2026-04-12

### Fixed
- Windows 证书自检新增 certutil 指纹兜底校验；即使扩展宿主里 PowerShell 读取 Root 存储偶发异常，也能正确识别当前用户已信任的本地 CA

## [0.1.31] - 2026-04-12

### Fixed
- 修复链路自检里 Windows 证书状态误报：当前用户 Root 证书存储已存在匹配指纹时，证书项将正确显示为正常

## [0.1.30] - 2026-04-12

### Added
- 高级设置新增“上游代理（可选）”输入框，可直接填写本地 HTTP/SOCKS 代理端口，避免用户手动改扩展配置文件

### Fixed
- Windows 证书自检改为分别读取 CA 文件指纹和当前用户 Root 存储指纹再做比较，避免 PowerShell 单行命令在部分环境下误报“证书未受信任”

## [0.1.29] - 2026-04-11

### Fixed
- 检测到 TUN/虚拟网卡时，若用户已在扩展中显式配置可复用的上游代理（如本地 SOCKS5 端口），扩展将允许继续接管而不再误阻断，解决“已有本地代理端口但仍提示已阻止接管”的问题

## [0.1.28] - 2026-04-11

### Fixed
- 透明代理接管前新增真实链路预演：只有本地 ai-monitor、候选上游代理和服务端 /health 都可用时，才会覆盖 VS Code 的 http.proxy，预演失败则保持现有网络链路不变
- 同一用户在不同 IDE（如 VS Code、Cursor、Insiders）下的本地待发队列、今日统计和运行态缓存改为按应用隔离，避免多个 IDE 实例互相覆盖状态

## [0.1.27] - 2026-04-11

### Fixed
- 修复“链路自检”里的证书安装误报：Windows 下改为按 ca.crt 的实际指纹校验当前用户 Root 证书存储，不再仅按主题名模糊匹配
- 证书检查改为使用系统 PowerShell 绝对路径，避免扩展宿主进程 PATH 差异导致的误判

## [0.1.26] - 2026-04-11

### Added
- 高级设置下新增“链路自检”，可直接查看本地代理、当前用户证书、上游代理和最近上报是否正常
- 链路自检会显示最近一次 collect / my-stats 同步时间、本地待发送队列和服务端健康状态，便于用户自助排查

## [0.1.25] - 2026-04-11

### Fixed
- 新安装扩展默认不再立即接管 VS Code 网络代理，避免与同事现有的桌面代理或系统代理直接冲突
- 首次启用监控代理时会先自动安装当前用户的本地 CA 证书，证书安装失败时不会继续接管网络，减少 HTTPS 证书错误
- 透明代理现在会额外继承 Windows 系统代理作为上游，兼容只配了系统代理而没有单独设置 VS Code http.proxy 的桌面代理环境

## [0.1.24] - 2026-04-11

### Fixed
- 监控面板里的“检查更新”增加明确反馈，手动点击后会显示检查中、已是最新版本、检查失败等结果，不再无感
- 手动检查更新不再复用静默逻辑，未配置上报地址、服务端异常或请求超时都会直接提示原因

## [0.1.15] - 2026-04-07

### Fixed
- 监控面板脚本改为 nonce 白名单执行，修复部分环境下 HTML 能显示但脚本未真正运行、导致整页无法点击的问题
- Webview CSP 收敛为 VS Code 推荐模式，避免脚本加载策略不一致造成的交互失效

## [0.1.14] - 2026-04-07

### Fixed
- 控制面板的按钮与折叠区改为脚本显式绑定点击事件，不再依赖内联 onclick，修复部分环境下整页“看得到但点不动”的问题
- 交互入口增加空节点保护，避免单个元素异常影响整页点击逻辑

## [0.1.13] - 2026-04-07

### Fixed
- 一键检查新增 Webview↔扩展握手确认（ACK），3 秒未收到扩展响应会直接提示“诊断通道异常”，避免无感卡住
- 扩展侧诊断增加总超时保护，超时会主动回传错误结果，不再出现前端长期等待

## [0.1.12] - 2026-04-07

### Fixed
- 监控面板新增前端防卡死机制：消息处理异常隔离、全局错误自动解锁、诊断看门狗超时回退
- 基础设置与身份提示相关节点增加空值保护，避免个别渲染异常导致整页交互失效

## [0.1.11] - 2026-04-07

### Fixed
- 修复“一键检查”在服务端无响应或异常时可能一直停留在“检查中”的问题，增加后端与前端双重超时兜底
- 修复身份校验请求无超时导致的面板状态卡住问题，基础设置变更后会在超时场景下明确提示“校验超时/不可用”

## [0.1.10] - 2026-04-07

### Added
- 监控面板增加“一键检查”，直接检查基础配置、本地代理、VS Code 代理路由、服务端健康状态、身份校验结果和最近代理日志，用于定位“用户已入库但没有 Token 记录”的原因

## [0.1.9] - 2026-04-07

### Fixed
- 状态栏同步显示“需重载”状态，用户即使不打开监控面板，也能知道当前窗口尚未进入 Token 采集链路
- 重载状态更新收敛为统一入口，避免弹窗、面板和状态栏提示不一致

## [0.1.8] - 2026-04-07

### Fixed
- 监控面板会明确显示“代理已运行但当前窗口待重载”的状态，并提供一键重载入口，避免用户关闭弹窗后仍看不出为什么没有 Token 上报
- 面板里手动启动代理时也纳入同一套重载提示逻辑，避免只修启动阶段而漏掉手动启用路径

## [0.1.7] - 2026-04-07

### Fixed
- 首次启用透明代理或修改代理路由后，扩展会明确提示“需要重载窗口”，避免只发 heartbeat / my-stats 但当前窗口里的 AI 请求仍未进入 /api/collect

## [0.1.6] - 2026-04-07

### Fixed
- 基础设置区增加工号身份校验提示，明确区分“新用户”“已匹配”“疑似重复”“工号冲突”等状态
- 同一工号在多个应用共用改为显式提示为正常情况，避免误判为重复配置
- 服务端阻止“重复工号 + 不同姓名”覆盖原用户信息，降低串号风险

## [0.1.5] - 2026-04-07

### Changed
- 监控面板的“刷新数据”增加可视化反馈，包含按钮 loading、统计卡片刷新态与最近同步时间，避免点击后无感

## [0.1.4] - 2026-04-07

### Fixed
- 开发版 dashboard 模板同步改为按当前员工统计刷新，避免调试视图仍然误读全局汇总
- 关闭扩展时不再触发未等待完成的强制 flush，离线队列改为稳妥落盘到下次会话继续补传
- 压低 wsClient 测试中的连接日志噪音，减少打包时的无关输出

## [0.1.3] - 2026-04-07

### Fixed
- 监控面板的今日 Tokens 和请求数改为按当前填写员工号同步，不再误显示服务器全局汇总
- “刷新数据”会先尝试补上报再重拉当前员工统计，修复新员工配置后刷新无感的问题
- 本地 todayStats 与离线队列改为按 serverUrl + userId 隔离，避免切换员工身份后串用旧缓存

## [0.1.2] - 2026-04-07

### Changed
- 监控面板改为更克制的文字化布局，移除火苗图标与竖排上报地址
- 重做头部身份卡、顶部信息条和设置区层级，使大窗口与小窗口下都更舒适

## [0.1.1] - 2026-04-07

### Changed
- 监控面板头部改为更紧凑的身份卡布局，移除独立“新会话”主按钮
- 离线状态下的身份卡改为红色高亮，便于快速识别异常

### Fixed
- 修复扩展 TypeScript 构建配置，补齐 Node、VS Code、Jest 类型声明，恢复正常编译与打包

## [0.1.0] - 2026-04-06

### Added
- 核心架构：EventBus 事件总线驱动的解耦设计
- @monitor Chat Participant - 在对话中自动记录 Token 用量
- 5 个 LM Tools（代码审查、解释、测试生成、文档生成、重构建议）
- Dashboard 监控面板 - 实时查看用量统计和多维度分析
- WebSocket 客户端 - 支持后端实时推送（心跳、断线重连）
- 数据收集器插件 - 支持 Cursor、Cline、Continue 扩展
- Copilot Metrics API 集成（支持多组织）
- 多维度统计（按应用、模型、来源分类）
- 离线队列 - 网络故障自动缓存和重试
- 多平台打包（Windows/macOS/Linux）
- GitHub Actions CI/CD 自动化
- Jest 单元测试框架

### Removed
- MITM 代理模式（已移除 Gateway 配置）
